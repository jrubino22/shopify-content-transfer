# Shopify File Transfer

A CLI tool to export images from one Shopify store and import them into another.

---

## Prerequisites

- Node.js 18+
- Permanent access tokens for both stores (use [shopify-token-generator](https://github.com/your-org/shopify-token-generator) to get them)

---

## Setup

### 1. Install dependencies
```bash
npm install
```

### 2. Configure your .env
```bash
cp .env.example .env
```

Fill in your store URLs and access tokens:
```env
FROM_STORE_URL=your-source-store.myshopify.com
FROM_ACCESS_TOKEN=shpca_xxxxxxxxxxxxxxxxxxxx

TO_STORE_URL=your-dest-store.myshopify.com
TO_ACCESS_TOKEN=shpca_xxxxxxxxxxxxxxxxxxxx
```

---

## Usage

### Export images from source store

Fetches all images from the source store and saves them to `export.json`.
```bash
node fileTransfer.js export
```

### Import images to destination store

Reads `export.json` and uploads all images to the destination store.
```bash
node fileTransfer.js import
```

### Transfer in one step

Runs export then import back to back.
```bash
node fileTransfer.js transfer
```

---

## How it works

**Export** paginates through the source store's Files API and saves each image's
URL and metadata to a local `export.json` file.

**Import** loops through `export.json` and for each image:
1. Requests a staged upload target from the destination store
2. Fetches the raw image bytes from the source URL
3. Uploads the bytes to the staged target
4. Registers the file in the destination store via the `fileCreate` mutation

---

## Files

| File | Description |
|------|-------------|
| `fileTransfer.js` | Main CLI script |
| `export.json` | Generated by export — list of images to import |
| `.env` | Your store URLs and access tokens |

---

## Notes

- `export.json` is safe to re-use — you can run `import` multiple times if
  needed without re-running `export`
- If an individual file fails to import it will be logged and skipped — the
  rest of the import will continue
- Shopify's API rate limits are respected with a short delay between uploads
