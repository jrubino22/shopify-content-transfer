# Shopify File Transfer

A CLI tool to transfer content between Shopify stores. Currently supports
images, metaobjects, and product/variant metafields.

---

## Prerequisites

- Node.js 18+
- Permanent access tokens for both stores (use [shopify-token-generator](https://github.com/your-org/shopify-token-generator) to get them)

---

## Setup

### 1. Install dependencies
```bash
npm install
```

### 2. Configure your .env
```bash
cp .env.example .env
```

Fill in your store URLs and access tokens:
```env
FROM_STORE_URL=your-source-store.myshopify.com
FROM_ACCESS_TOKEN=shpca_xxxxxxxxxxxxxxxxxxxx

TO_STORE_URL=your-dest-store.myshopify.com
TO_ACCESS_TOKEN=shpca_xxxxxxxxxxxxxxxxxxxx
```

## Usage

Each transfer type is its own file and can be run independently.

### Images
```bash
node fileTransfer.js export       Fetch all images from source store â†’ export.json
node fileTransfer.js import       Upload images from export.json â†’ dest store
node fileTransfer.js transfer     Do both in one step
```

### Metaobjects
```bash
node metaobjectTransfer.js export      Export definitions + entries â†’ metaobjects_export.json
node metaobjectTransfer.js import      Import from metaobjects_export.json â†’ dest store
node metaobjectTransfer.js transfer    Do both in one step
```

### Metafields (products + variants)
```bash
node metafieldTransfer.js export      Export definitions + values â†’ metafields_export.json
node metafieldTransfer.js import      Import from metafields_export.json â†’ dest store
node metafieldTransfer.js transfer    Do both in one step
```

---

## Behavior

**All transfer scripts follow the same rules:**

- If a definition or entry already exists in the destination store it will be **skipped**, not overwritten
- If a mismatch is detected between source and destination data structures, it will be **logged as a warning** and the operation will continue
- A **summary** is printed at the end of every import showing counts of created, skipped, warned, and failed records

## Export files

Each script generates its own export file which can be reused â€” you can run
`import` multiple times without re-running `export`.

| File | Generated by |
|------|-------------|
| `export.json` | `fileTransfer.js export` |
| `metaobjects_export.json` | `metaobjectTransfer.js export` |
| `metafields_export.json` | `metafieldTransfer.js export` |

## Required app scopes

Two separate Shopify apps are required â€” one installed on each store.

**Source store app:**
- `read_files`
- `read_metaobject_definitions`
- `read_metaobjects`
- `read_products`

**Destination store app:**
- `write_files`
- `write_metaobject_definitions`
- `write_metaobjects`
- `write_products`

---

## Notes

- Metafield definitions are always imported before values to ensure type
  validation is in place before any data is written
- Metafields using the `custom` namespace are handled automatically â€” the
  `access` field is omitted for merchant-owned namespaces to avoid API errors
- Shopify API rate limits are respected with short delays between requests
- All export files are safe to commit to version control as they contain no
  credentials
